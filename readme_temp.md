# Middara Helper

Companion-приложение для настольной игры **Middara** на Android. Позволяет отслеживать состояние героев, управлять карточками снаряжения, следить за очками выносливости и здоровья, разряжать карты и фиксировать конец хода.

## Возможности

- **Несколько героев** — создавайте и называйте произвольное количество героев, каждый хранит свой полный набор карт, который при желании можно скрыть
- **Планшет героя** — базовые характеристики (ПЗ, АТК, ЗЩ, СКР, Слоты) отображаются рядом с карточкой
- **Снаряжение** с вложенной логикой:
  - Одноручное × 2 **или** двуручное × 1
  - К каждому оружию можно добавить **улучшение оружия**
  - Броня + **улучшение брони**
  - Диковинка
- **Реликвии** (до 3 шт.) и **Аксессуар** (1 шт.) — отдельные секции
- **Рюкзак** (до 3 предметов), **Карты дисциплин** (до 8), **Одноразовые предметы**, **Спутники** (фамильяры и компаньоны)
- **Очки здоровья и повреждения** — счётчики с кнопками + и −
- **ПВ (пункты выносливости)** — 5 кружков, клик по кружку сразу устанавливает нужное значение; повторный клик сбрасывает в 0
- **Разрядить / Зарядить** — поворот карты на 90°, повторное нажатие возвращает в исходное положение
- **Перевернуть** — зеркальное отражение карты по горизонтали
- **Конец хода** — сбрасывает все разряженные карты обратно в исходное положение ("перевернуть" не затрагивает)
- **Бонусы снаряжения** — суммарные характеристики всех экипированных карт отображаются рядом с планшетом героя
- **Сворачиваемые секции** — все крупные категории (Снаряжение, Реликвии, Аксессуар, Рюкзак, Дисциплины, Одноразовые предметы, Спутники) сворачиваются/раскрываются по нажатию на заголовок
- **Поиск карт** — фильтрация по названию и описанию внутри браузера карт
- **Сохранение состояния** — всё хранится локально в базе данных Room, переживает перезапуск приложения

## Ключевые файлы

### `data/model/Card.kt`
Базовая модель данных для карты:
```kotlin
data class Card(
    val id: String,          // уникальный идентификатор
    val name: String,        // название карты
    val type: CardType,      // тип карты (WEAPON, ARMOR, DISCIPLINE и т.д.)
    val imageAsset: String,  // путь к PNG внутри assets/
    val description: String, // текстовое описание
    val stats: CardStats     // характеристики: hp, attack, defense, speed, slots
)
```

### `data/model/CardType.kt`
Перечисление всех типов карт с лимитами:

| Константа        | Отображаемое имя   | Макс. в руке |
|------------------|--------------------|:------------:|
| `HERO_SHEET`     | Планшет героя      | 1            |
| `WEAPON`         | Оружие             | 2            |
| `WEAPON_UPGRADE` | Улучшение оружия   | 2            |
| `ARMOR`          | Броня              | 1            |
| `ARMOR_UPGRADE`  | Улучшение брони    | 1            |
| `TRINKET`        | Диковинка          | 1            |
| `RELIC`          | Реликвия           | 3            |
| `ACCESSORY`      | Аксессуар          | 1            |
| `ITEM_UPGRADE`   | Улучшение предмета | 2            | (не бывает)
| `BACKPACK`       | Предмет рюкзака    | 3            |
| `DISCIPLINE`     | Карта дисциплины   | 8            |
| `CONSUMABLE`     | Одноразовый предмет| 10           |
| `FAMILIAR`       | Фамильяр           | 1            |
| `COMPANION`      | Компаньон          | 1            |

### `data/model/HeroHand.kt`
Всё состояние одного героя: имя, очки здоровья, урон, ПВ, режим оружия, все слоты карт. Содержит расширения:
- `totalBonusStats()` — суммирует характеристики всех экипированных карт
- `findEquippedCard(instanceId)` / `updateEquippedCard()` / `removeEquippedCard()` — управление слотами
- `resetAllDischargedCards()` — сброс всех повёрнутых карт в 0° (Конец хода)

### `data/repository/CardRepository.kt`
Читает `assets/data/cards.json` при первом обращении (lazy). Предоставляет:
- `allCards` — все карты
- `getByType(CardType)` — карты конкретного типа
- `searchByType(query, CardType)` — поиск с фильтром по типу

### `ui/viewmodel/HandViewModel.kt`
Весь игровой движок руки героя: установка карт, переключение режима оружия, разряд/flip, конец хода, изменение HP/урона/ПВ. Сохраняет состояние в Room через `HeroHandRepository`.

### `ui/screens/HeroHandScreen.kt`
Главный игровой экран. Отображает всю руку героя построчно: токены → планшет → снаряжение → реликвии → аксессуар → рюкзак → дисциплины → расходники → спутники. Открывает `CardBrowserSheet` / `TabbedCardBrowserSheet` для выбора карт.

### `ui/screens/CardBrowserSheet.kt`
Нижний лист для выбора карт. Два режима:
- `CardBrowserSheet` — одиночный выбор по типу (с поиском)
- `TabbedCardBrowserSheet` — с вкладками (например, «Оружие 1» + «Улучшение оружия»); галочка на вкладке если карта уже выбрана

### `ui/components/CardView.kt`
Отрисовка карты: изображение из assets или цветная заглушка с названием. Показывает угловые бейджи при повороте и перевороте. Содержит:
- `CardStatsSummary` — характеристики одной карты
- `EquipmentBonusSummary` — суммарные бонусы (зелёный `+`, красный `−`)
- `cardTypeColor()` — цвет карты по типу (настраивается в `Color.kt`)

### `ui/theme/Color.kt`
Все цвета приложения, включая цвета фона карт по типу: `CardColorWeapon`, `CardColorArmor`, `CardColorDiscipline`, `CardColorRelic` и т.д. Меняйте здесь, чтобы изменить цвет заглушек карт на экране.

## Наполнение контентом

### `cards.json` — база карт

**Путь:** `app/src/main/assets/data/cards.json`

Это единственный файл, который нужно редактировать для добавления карт. Формат — массив JSON-объектов:

```json
[
  {
    "id": "hero_rook",
    "name": "Рук",
    "type": "HERO_SHEET",
    "imageAsset": "images/cards/hero_rook.png",
    "description": "Могучий воин в тяжёлой броне.",
    "stats": {
      "hp": 14,
      "attack": 4,
      "defense": 4,
      "speed": 2,
      "slots": 2
    }
  },
  {
    "id": "weapon_iron_sword",
    "name": "Железный меч",
    "type": "WEAPON",
    "imageAsset": "images/cards/weapon_iron_sword.png",
    "description": "Надёжный одноручный меч.",
    "stats": {
      "hp": 0,
      "attack": 3,
      "defense": 0,
      "speed": 0,
      "slots": 1
    }
  }
]
```

**Правила полей:**

| Поле            | Тип    | Обязательно | Описание |
|-----------------|--------|:-----------:|----------|
| `id`            | string | да          | Уникальный идентификатор. Латиница, цифры, подчёркивание. |
| `name`          | string | да          | Отображаемое название карты |
| `type`          | string | да          | Одна из констант `CardType` строго в верхнем регистре: `WEAPON`, `ARMOR`, `DISCIPLINE` и т.д. |
| `imageAsset`    | string | нет         | Путь к PNG внутри папки `assets/`. Если файла нет — отображается цветная заглушка |
| `description`   | string | нет         | Текстовое описание карты |
| `stats.hp`      | int    | нет         | Бонус к здоровью (может быть отрицательным) |
| `stats.attack`  | int    | нет         | Бонус к атаке |
| `stats.defense` | int    | нет         | Бонус к защите |
| `stats.speed`   | int    | нет         | Бонус к скорости |
| `stats.slots`   | int    | нет         | Количество слотов снаряжения |

> **Важно:** значение поля `type` должно точно совпадать с константой из `CardType.kt`. Регистр имеет значение (`WEAPON`, не `weapon` или `Weapon`).

### Типы карт

```
HERO_SHEET      — планшет героя с базовыми характеристиками
WEAPON          — оружие (одноручное или двуручное)
WEAPON_UPGRADE  — улучшение оружия (появляется под слотом оружия)
ARMOR           — броня
ARMOR_UPGRADE   — улучшение брони (появляется под слотом брони)
TRINKET         — диковинка
RELIC           — реликвия (до 3 шт.)
ACCESSORY       — аксессуар (1 шт.)
ITEM_UPGRADE    — улучшение предмета (общий тип)
BACKPACK        — предмет рюкзака
DISCIPLINE      — карта дисциплины (навыки, до 8 шт.)
CONSUMABLE      — одноразовый предмет
FAMILIAR        — фамильяр (спутник)
COMPANION       — компаньон (спутник)
```

### Изображения карт

**Путь:** `app/src/main/assets/images/cards/`

- Формат: **PNG**
- Рекомендуемый размер: **450 × 630 пикселей** (соотношение сторон 5:7)
- Прозрачность поддерживается
- Имя файла должно совпадать с полем `imageAsset` в `cards.json`

Если изображение отсутствует или не загружается — приложение автоматически показывает цветную заглушку с названием карты. Цвет заглушки определяется типом карты (настраивается в `ui/theme/Color.kt`).

**Пример соответствия:**
```
cards.json → "imageAsset": "images/cards/hero_rook.png"
файл       → app/src/main/assets/images/cards/hero_rook.png
```

## Архитектура

```
UI (Compose)
  └── HeroHandScreen / HeroListScreen
        └── HandViewModel (StateFlow<HeroHand>)
              ├── HeroHandRepository
              │     └── AppDatabase (Room)  ← сохранение состояния героя
              └── CardRepository
                    └── cards.json (assets) ← база карт (только чтение)
```

- **Однонаправленный поток данных:** ViewModel хранит `StateFlow<HeroHand>`, экран только читает состояние и вызывает команды
- **Карты — иммутабельны:** `cards.json` читается один раз при запуске, никогда не изменяется в рантайме
- **Состояние героя — персистентно:** каждое изменение немедленно сохраняется в Room (`HeroHandEntity` хранит JSON-сериализованный `HeroHand`)
- **Изображения — lazy:** загружаются через Coil из `assets/` по требованию, при ошибке отображается заглушка
